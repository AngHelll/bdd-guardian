# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability in BDD Guardian,
please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities.
2. Email your findings to: **angel.jimenez@gbm.com**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours of your report
- **Status Update**: Within 7 days with our assessment
- **Resolution**: We aim to fix critical issues within 30 days

### Scope

This policy applies to:
- The BDD Guardian VS Code extension
- The source code in this repository

### Out of Scope

- Vulnerabilities in VS Code itself
- Vulnerabilities in third-party dependencies (please report to the maintainers)

## Security Best Practices

When using BDD Guardian:
- Keep the extension updated to the latest version
- Only install from the official VS Code Marketplace or trusted sources
- Review extension permissions before installation

Thank you for helping keep BDD Guardian secure!
